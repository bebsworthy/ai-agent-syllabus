# Warp and Oz: Getting Started Guide

**Source:** https://www.warp.dev/university?slug=university (redirects to https://docs.warp.dev/)

## Overview

Warp is described as "an **Agentic Development Environment**, a modern terminal combined with powerful agents" that facilitate code development tasks. The platform integrates with Oz, which functions as "the orchestration platform for cloud agents."

## Warp Terminal Features

The Warp application provides developers with several key capabilities:

- **Agent Modality**: Users can toggle between a standard terminal interface and a dedicated chat view for extended agent conversations
- **Modern Terminal Experience**: Built with Rust for performance, featuring cursor navigation, block-based editing, multiline support, and syntax highlighting
- **Integrated Code Editor**: Includes file browsing, LSP-powered code editing, and collaborative review tools
- **Agent Compatibility**: Works with various coding assistants, supporting voice interaction and specialized terminal features

## Oz Platform Capabilities

Oz operates in two distinct modes:

### Local Agents
Interactive agents running within Warp that handle real-time coding support. They can refactor code, diagnose and resolve bugs, execute terminal commands, and orchestrate complex workflows while keeping developers in control through review and approval workflows.

### Cloud Agents
Background-operating agents that automate repetitive tasks through:
- Event-triggered reactions (Slack, Linear, GitHub, webhooks)
- Scheduled recurring jobs
- Concurrent multi-repository processing
- Complete audit trails and team visibility

## Integration and Workflow

The platforms work cohesively, enabling developers to transition between local interactive work and cloud-based automation without losing context or progress. Shared team knowledge bases and tool integrations remain consistent across both environments.

## Model Selection and Privacy

Oz supports multiple LLM providers, allowing teams to select optimal models per task. The platform maintains SOC 2 compliance with zero-retention data policies across all contracted providers.
