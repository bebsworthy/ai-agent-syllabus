# Code Reviews: Just Do It

**Source:** https://blog.codinghorror.com/code-reviews-just-do-it/

---

## Overview

In referencing Karl Wiegers' work on peer reviews, the author emphasizes that peer examination of code is "one of the most powerful software quality tools available."

Based on experience at Vertigo, the argument is presented that **code reviews represent the single most impactful action teams can take to enhance code quality**. Code remains incomplete until reviewed by another developer.

## Evidence from Code Complete

Steve McConnell's research demonstrates superior results for inspections compared to testing alone:

- **Unit testing effectiveness:** 25%
- **Function testing effectiveness:** 35%
- **Integration testing effectiveness:** 45%
- **Code inspections effectiveness:** 60%

### Real-world Case Studies

Dramatic improvements have been documented across organizations:

- A software maintenance organization reduced single-line change errors from 55% to 2% after implementing reviews.
- Another group saw error rates drop from 4.5 per 100 lines of code to 0.82—an 80% reduction—when reviews were introduced.
- IBM's 500,000-line Orbit project employed 11 inspection levels and delivered early with approximately 1% of expected errors.
- AT&T documented a 14% productivity increase and 90% defect decrease following review implementation.

## Recommendation

For organizations new to code reviews, *Peer Reviews in Software* by Karl Wiegers is recommended as a practical guide.
