# Week 6: AI Testing and Security

**Dates:** Mon 10/27 & Fri 10/31

---

## Objectives

Understand the security implications of AI-assisted development — prompt injection, vibe coding risks, and vulnerability detection. Learn how SAST and DAST tools are being supercharged with AI, and how to write secure code in the age of LLMs. Guest: Isaac Evans, CEO of Semgrep.

> **Note:** This folder contains previously collected materials from the "Week 5 - Security" folder. The course website lists this as Week 6.

---

## Topics

- Secure vibe coding — the risks of AI-generated code and how to mitigate them
- History of vulnerability detection — from static analysis to AI-powered scanning
- AI-generated test suites — automated quality assurance at scale
