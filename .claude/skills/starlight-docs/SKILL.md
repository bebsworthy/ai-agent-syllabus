# Starlight Documentation Authoring

Reference guide for writing high-quality `.md` pages for Starlight (https://starlight.astro.build).
Use markdown only — no MDX unless the project already uses it.

---

## Frontmatter

Every page **must** have `title`. All other fields are optional.

```yaml
---
title: Page Title                          # required — shown at top, in browser tab, metadata
description: One-sentence summary.         # used by search engines and social previews
sidebar:
  label: Short Label                       # override long title in sidebar nav
  order: 2                                 # lower = higher in autogenerated groups
  badge:
    text: New                              # visual indicator on sidebar link
    variant: tip                           # note | tip | danger | caution | success
  hidden: false                            # true = exclude from sidebar entirely
template: doc                              # doc (default) | splash (full-width landing page)
tableOfContents:
  minHeadingLevel: 2                       # default: 2
  maxHeadingLevel: 3                       # default: 3
# tableOfContents: false                   # disable TOC entirely
prev: false                                # hide previous page link
next:
  label: Next Step
  link: /path/to/next
pagefind: true                             # false = exclude from search index
draft: false                               # true = excluded from production builds
editUrl: false                             # false = hide edit link; or override URL
lastUpdated: 2024-01-15                    # or true/false to override global setting
banner:
  content: '⚠️ This page is in beta.'     # announcement banner (supports HTML)
---
```

### Landing pages (`template: splash`)

Use `template: splash` with a `hero` block for index/home pages:

```yaml
---
title: My Product
template: splash
hero:
  title: Welcome to My Product
  tagline: The fastest way to do X.
  image:
    file: ../../assets/hero.png
  actions:
    - text: Get Started
      link: /getting-started/
      icon: right-arrow
      variant: primary
    - text: View on GitHub
      link: https://github.com/org/repo
      icon: github
      variant: minimal
---
```

## Content Structure Rules

- **Never use `# H1`** in the body — Starlight renders the frontmatter `title` as h1 automatically.
- Start body content with a paragraph (no heading immediately after frontmatter).
- Use `## H2` for main sections and `### H3` for subsections.
- H2 and H3 headings automatically appear in the page TOC.

---

## Asides (Callouts)

Use `:::type` syntax for callout boxes. Four built-in types:

```markdown
:::note
Neutral information the reader should be aware of.
:::

:::tip
Helpful advice or a shortcut.
:::

:::caution
Warn about something that could cause problems.
:::

:::danger
Critical warning — data loss, security risk, etc.
:::
```

**Custom title:**
```markdown
:::tip[Pro tip]
You can override the default title with square brackets.
:::
```

**Custom icon** (use any Starlight icon name):
```markdown
:::note{icon="information"}
Custom icon on a note.
:::
```

## Icons

Icons appear in three places in markdown/frontmatter:
1. Aside custom icons: `:::tip{icon="rocket"}`
2. Hero action buttons: `icon: right-arrow` in frontmatter
3. Social links in `astro.config.mjs`: `icon: 'github'`

### Icon names by category

**Navigation & arrows**
`right-arrow` `left-arrow` `up-arrow` `down-arrow`
`right-caret` `left-caret` `up-caret` `down-caret`

**Status & feedback** *(most useful in asides)*
`information` `warning` `error` `approve-check` `approve-check-circle`
`rocket` `star` `magnifier` `bars` `close`

**Social & community**
`github` `gitlab` `bitbucket` `discord` `twitter` `mastodon`
`linkedin` `youtube` `instagram` `reddit` `twitch` `facebook`
`email` `rss` `threads`

**Technology & platforms**
`astro` `node` `deno` `vscode` `npm` `git` `docker` `kubernetes`
`vercel` `netlify` `cloudflare`

**File-type icons** *(seti: prefix)*
`seti:javascript` `seti:typescript` `seti:python` `seti:rust` `seti:go`
`seti:json` `seti:css` `seti:markdown` `seti:html` `seti:yaml`

Full list: https://starlight.astro.build/reference/icons/

## Code Blocks

Use triple backticks with a language identifier. Starlight uses Expressive Code for rendering.

````markdown
```javascript
const x = 1;
```
````

**Add a filename/title:**
````markdown
```javascript title="src/index.js"
const x = 1;
```
````

**Highlight specific lines:**
````markdown
```javascript {2-3}
const a = 1;
const b = 2;   // highlighted
const c = 3;   // highlighted
```
````

**Mark insertions and deletions:**
````markdown
```diff
- const old = true;
+ const new = false;
```
````

Or with explicit markers:
````markdown
```javascript ins={2} del={1}
const old = 'removed';
const new = 'added';
```
````

**Shell/terminal commands** render with a terminal frame automatically:
````markdown
```bash
npm install @astrojs/starlight
```
````

## Links

Standard markdown links work for both internal and external:

```markdown
[Internal page](/guides/getting-started/)
[External site](https://example.com)
[Section anchor](#heading-text)
```

Internal links use the root-relative path from `src/content/docs/`.

## Images

```markdown
![Alt text describing the image](../../assets/screenshot.png)
```

- Always include descriptive alt text for accessibility.
- Use relative paths for local assets stored in `src/assets/`.
- Starlight uses Astro's optimized asset handling.

## Sidebar Configuration (astro.config.mjs)

Sidebar is configured in `astro.config.mjs`, not in markdown files.
Frontmatter `sidebar.*` fields only affect autogenerated groups.

### Autogenerate from a directory

```javascript
sidebar: [
  {
    label: 'Guides',
    autogenerate: { directory: 'guides' },
  },
]
```

### Manual links and groups

```javascript
sidebar: [
  { slug: 'getting-started' },
  {
    label: 'Reference',
    items: [
      { slug: 'reference/config' },
      { slug: 'reference/cli', badge: { text: 'New', variant: 'tip' } },
    ],
  },
  {
    label: 'External',
    link: 'https://example.com',
    attrs: { target: '_blank' },
  },
]
```

### Collapsed groups

```javascript
{
  label: 'Advanced',
  collapsed: true,
  autogenerate: { directory: 'advanced' },
}
```

## Quality Checklist

Before finishing any doc page:

- [ ] `title` is set in frontmatter
- [ ] `description` is set (1–2 sentences, used by search engines)
- [ ] No `# H1` in the body
- [ ] Body starts with a paragraph, not a heading
- [ ] All images have descriptive alt text
- [ ] Code blocks have a language identifier
- [ ] Asides use `:::type` syntax, not blockquotes
- [ ] `sidebar.label` is set if the title is too long for the nav
- [ ] `sidebar.order` is set if the page needs a specific position
- [ ] `draft: true` is set for work-in-progress pages
- [ ] `pagefind: false` is set for pages that should not appear in search

## Common Patterns

### Getting started / tutorial page

```yaml
---
title: Getting Started
description: Install and configure X in under 5 minutes.
sidebar:
  order: 1
---

Brief intro paragraph.

## Prerequisites

- Node.js 18+
- ...

## Installation

```bash
npm install x
```

## Next Steps

- [Configuration](/config/)
- [API Reference](/reference/)
```

### Reference page

```yaml
---
title: Configuration Reference
description: All available configuration options for X.
tableOfContents:
  maxHeadingLevel: 4
sidebar:
  label: Configuration
---
```

### Changelog / release notes (exclude from search)

```yaml
---
title: Changelog
pagefind: false
tableOfContents: false
---
```
